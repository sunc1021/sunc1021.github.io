1.封装

所谓封装，就是把客观事物封装成抽象概念的类，并且类可以把自己的数据和方法只向可信的类或者对象开放，向没必要开放的类或者对象隐藏信息。

实现封装就是控制类或成员的可见性范围。这就需要依赖访问控制修饰符，也称为权限修饰符来控

1.1成员变量私有化

概述：私有化类的成员变量，提供公共的get和set方法，对外暴露获取和修改属性的功能。

例子

1.2成员变量封装优点

  • 让使用者只能通过事先预定的方法来 访问数据 ，从而可以在该方法里面加入控制逻辑，限制对成员变量的不合理访问。还可以进行数据检查，从而有利于保证对象信息的完整性。
  • 便于修改 ，提高代码的可维护性。主要说的是隐藏的部分，在内部修改了，如果其对外可以的访问方式不变的话，外部根本感觉不到它的修改。例如：Java8->Java9，String从char[]转为byte[]内部实现，而对外的方法不变，我们使用者根本感觉不到它内部的修改。

注意：

  • 开发中，一般成员实例变量都习惯使用private修饰，再提供相应的public权限的get/set方法访问。
  • 对于final的实例变量，不提供set()方法。（后面final关键字的时候讲）
  • 对于static final的成员变量，习惯上使用public修饰。

2.继承

概念：类与类之间属性和方法的赠与(父类)和获得(子类)

2.1作用

​ 提高代码的重复性

2.2语法

​ public class 子类 extends 父类{}

2.3特点

  1. 子类继承父类，子类可以继承到父类的属性和方法

  2. 子类继承父类，子类可以定义独有的属性和方法

  3. 在java中只能是单继承，及一个子类只能有一个父类

  4. 在java中支持多层继承，及父类还可以有自己父类，java中最顶层的父类是Object

    (所有没有显示的写extends继承默认都是Object类的子类)

  5. 不能被继承

    1. 私有的属性和方法不能被继承
    2. 构造方法不能被继承
    3. 使用默认修饰符修饰属性和方法在不同包中不能被继承

  6. 继承中：

    • 成员变量访问特点：看等号左边是谁，先调用谁中的成员变量
    • 成员方法访问特点：看new的是谁，先调用谁中的方法

3.方法重写

父类的所有方法子类都会继承，但是当某个方法被继承到子类之后，子类觉得父类原来的实现不适合于自己当前的类，该怎么办呢？子类可以对从父类中继承来的方法进行改造，我们称为方法的重写(override、overwrite)。也称为方法的重置、覆盖。

前提：继承

访问：看new的是谁，先调用谁中，如果new的是子类，调用子类的重写方法，子类没有，找父类

检测是否为重写方法：在方法上写@Override

3.1注意事项

  1. 子类重写父类方法后，权限必须要保证大于或等于父类权限（权限指的是访问权限）
    public > protected > 默认 > private
  2. 子类方法重写父类方法，方法和参数名要一样
  3. 私有方法、构造方法、静态方法不能重写
  4. 父类的返回值时基本数据类型，则子类重写方法的返回值类型必须相同
  5. 父类的返回值时引用数据类型，子类重写父类方法后，返回值类型应该是父类方法返回值类型的子类类型
  6. （超纲）子类重写的方法抛出的异常类型可以与父类被重写的方法抛出的异常类型相同，或是父类被重写方法抛出异常的子类

4.super和this

4.1继承中构造方法的特点

  1. new子类对象是，会先初始化父类（先走父类的无参构造）
  2. 每个构造方法的第一行，默认会有一个super()
  3. 若父类和子类有相同属性名的属性，则是两个不同的属性，遵从就近原则。规范要求避免相同命名，可读性降低

4.2super的具体使用

  1. 概述：代表父类引用
  2. 作用：可以调用父类的成员
  3. 使用：在第一行使用
    1. 调用父类构造方法：在子类的构造中写
      super()——>调用父类无参构造
      super(实参)——>调用父类有参构造
    2. 调用父类成员变量
      super.成员变量
    3. 调用父类成员方法
      super.成员方法

4.3this的具体使用

  1. 概念：代表当前的对象

  2. 作用

    1. 区分重名的成员变量和局部变量
    2. 调用当前对象中的成员

  3. 使用：

    1. 调用当前对象的构造：在构造里写

      this()——>调用当前对象无参构造
      this(实参)——>调用当前 对象有参构造

    2. 调用当前对象的成员变量
      this.成员变量名

    3. 调用当前对象的成员方法
      this.成员方法名（实参）

  4. 注意：
    不管super还是this，只要在构造方法中使用，都必须在第一行，所以二者不能同时写出来

4.4继承特点

1.继承只支持单继承，不能多继承

2.支持多层继承

3.一个父类可以又多个子类

4.

  • 构造方法不能继承，不能重写
  • 私有方法可以继承，但不能重写
  • 静态方法可以继承，但不能重写

5.多态

  • 不要从字面意思理解这两个字，要从使用形式上掌握
  • 要知道多态的好处
  • 要知道多态的前提

5.1多态介绍

  1. 前提
    1. 必须有字符类继承或接口实现关系
    2. 必须有方法的重写（没有重写，多态没有意义），多态主要完重写方法
    3. new对象：父类引用指向子类对象
      Fu fu = new Zi( ) ——>理解为大类型接受一个小类型数据，如double a = 10
  2. 多态不能调用子类特有功能

5.2多态的基本使用

5.3多态下成员访问特点

成员变量：看等号左边是谁，先调用谁中的变量

成员方法：看new的是谁，先调用谁中的方法

5.4多态的好处

1.问题描述

如果使用原始方式new对象（等号左右一致），即能调用重写的，还能调用继承的，还能调用自己特有的成员

但多态方式new对像，只能调用重写的，那为社么还用多态呢？

2.原始方式好处

  1. 优点：即能调用重写的，还能调用继承的，还能调用自己特有的成员
  2. 缺点：扩展性差

3.多态的好处

  1. 优点：扩展性强
  2. 缺点：不能直接调用子类特有功能

5.5多态中的转型

5.5.1向上转型

父类引用指向子类对象

父类类型 对象名 = new 子类对象（）

如：double b = 10；

5.5.2向下转型

将大类型强制转为小类型，好比强制类型转换
父类类型 对象名1 = new 子类对象（）-> 向上转型 -> double b=1

子类类型 对象2 = (子类类型)对象1 -> 向下转型 -> int i=(int)b

5.6转型可能会出现问题

  1. 如果等号左右两边类型不一致，会出现类型转换异常
  2. 向下转型之前，先判断类型：instanceof，判断结果为boolean
  3. 对象名 instanceof 类型 -> 判断的是关键字前面的对象是否符合关键字后面的类型
